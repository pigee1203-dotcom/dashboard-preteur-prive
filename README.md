# Dashboard prêteur privé

Ce projet vise à remplacer les outils bancaires complexes par un dashboard
simple, clair et adapté aux prêteurs privés.

## Problème actuel
Les logiciels comme Margill mélangent :
- le contrat de prêt
- la souffrance (NSF, reports)
- les frais

Ce mélange rend les calculs incompréhensibles et force des ajustements manuels.

## Objectif
Créer un système où :
- le contrat est stable
- la souffrance est séparée
- chaque action est simple et traçable

## Règle d’or
Le contrat ne change jamais.
La souffrance est une couche séparée.
